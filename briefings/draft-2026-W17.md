This week's Grid-the-Grey CISO AI Security Brief — the attack surface has moved decisively into your AI toolchain, and this week proves it's no longer theoretical.

Start with Flowise. CVE-2025-59528 is a CVSS 10.0 remote code execution flaw in the Flowise AI agent builder, actively exploited right now against over twelve thousand internet-exposed instances. The vulnerability sits in the CustomMCP node and allows unauthenticated JavaScript execution with full Node.js privileges. No authentication required. This maps directly to MITRE ATLAS AML.T0047 and OWASP LLM07 — insecure plugin design giving attackers a direct path into your agentic infrastructure. If your teams are running Flowise, treat this as a fire drill today.

That Flowise flaw connects to a deeper architectural problem. Researchers have confirmed a structural vulnerability in Anthropic's Model Context Protocol — MCP — that allows unsanitised commands to execute silently across AI environments. The critical word here is "by design." This isn't a bug that gets patched in a point release. It's a protocol-level decision that enables supply chain compromise at scale. Your AI agents trusting MCP integrations are potentially trusting a compromised channel.

The supply chain theme runs through the week. The litellm Python library — version 1.82.8 — was found shipping a hidden file that executed arbitrary code on every Python interpreter startup. Any developer or AI pipeline touching that package was silently compromised. Separately, a tool called Gas Town with over fourteen thousand GitHub stars appears to have been consuming users' LLM API credits and GitHub permissions to do work for its own maintainers — without consent. These aren't sophisticated nation-state operations. These are dependency risks your teams are onboarding through normal development workflows.

Then there's the coding agent problem. Claude Code, Gemini CLI, and GitHub Copilot Agents are all vulnerable to prompt injection via malicious source code comments — a technique called Comment and Control. Cursor AI had a chained exploit combining prompt injection with a sandbox escape to achieve full shell access on developer machines. Your developers are using these tools daily. The attack surface is their workstations.

The pattern this week is convergence. Prompt injection, supply chain compromise, and excessive agent privilege are no longer separate categories — attackers are chaining them. AI tools are becoming the preferred lateral movement vector.

Three actions for this week. First, audit every internet-facing Flowise deployment immediately and isolate until patched. Second, inventory all LLM-adjacent Python dependencies and cross-reference against known compromised versions, starting with litellm. Third, establish a policy requiring explicit review of any developer tool that requests LLM API credentials or cloud account permissions before installation is permitted.
